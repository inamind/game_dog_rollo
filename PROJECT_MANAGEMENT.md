# Rollo Game 프로젝트 관리

## 이미지 에셋 제작 규칙

### 파일 형식과 배경

- 게임에 사용하는 모든 캐릭터 이미지는 실제 알파 채널이 포함된 투명 배경 PNG로 제작한다.
- 투명 배경처럼 보이는 체크무늬나 단색 배경을 이미지에 포함하지 않는다.
- 완성된 파일은 저장 후 알파 채널과 모서리 픽셀의 투명 여부를 확인한다.

### 캐릭터 크기와 메모리 사용

- 게임의 지원 화면 해상도는 최소 HD(1280×720)부터 최대 FHD(1920×1080)까지로 한다.
- 화면에 렌더링되는 캐릭터의 면적은 모든 지원 해상도에서 전체 화면 면적의 1/20보다 작아야 한다.
- 원본 캐릭터 이미지는 최대 해상도인 FHD에서 선과 질감이 미려하게 보이도록 FHD 기준으로 제작한다.
- FHD 화면 면적은 2,073,600px이므로 원본 한 프레임의 이미지 캔버스는 103,680px 미만으로 유지한다. 기본 권장 크기는 320×320px 이하이다.
- HD 화면에서는 원본 이미지를 다시 제작하지 않고 런타임에 비례 축소한다. HD 화면 면적은 921,600px이므로 실제 렌더링 면적을 46,080px 미만으로 제한한다.
- 화면 해상도가 달라질 때 캐릭터는 동일한 화면 면적 비율을 유지하도록 비례 조정하며, 확대보다 축소를 우선하여 화질 저하를 방지한다.
- 불필요한 투명 여백을 최소화하여 파일 크기, 메모리 사용량 및 로딩 시간을 줄인다.
- 같은 동작의 모든 프레임은 캐릭터의 비율, 기준 위치, 캔버스 크기를 일관되게 유지한다.

### 모션 이미지

- 각 요청 동작은 자연스럽게 이어지는 PNG 이미지 3프레임으로 제작한다.
- 모션 이미지는 `image_sources` 폴더에 저장한다.
